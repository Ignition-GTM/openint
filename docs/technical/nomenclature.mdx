## Nomenclature

Used in a sentence:
A <strong>Connection</strong> simply connects a data <strong>Resource</strong> to/from Venice which transmits data across the <strong>Pipeline</strong> protocol, storing any data in Venice's unified <strong>Database</strong> and making it available either via an <strong>API</strong> or to <strong>Export</strong>. 


| Term                              | Description                                       | Example                     |
| :-------------------------------- | :------------------------------------------------ | :-------------------------- |
| <strong>Resource</strong> | A generic resource which can eventually be used as a <strong>Source</strong> and/or a <strong>Destination</strong> within a <strong>Connection</strong> (e.g. kept agnostic of how it is connected within Venice) | <ul><li>You authenticate your bank with Venice, and eventually can choose whether to read data from that resource</li><li>You authenticate Google Sheets with Venice and can eventually choose whether to read/write to or from a spreadsheet</li></ul> |
| <strong>Connection</strong> | A connection (relationship) between Venice and a <strong>Resource</strong>, either as a <strong>Source</strong> or <strong>Destination</strong> of Venice data | <ul><li>A bank account associated with a <strong>Resource</strong> is selected as a <strong>Source</strong> of data for Venice to pull from</li><li>A table within a Coda document is selected as a <strong>Destination</strong> to write data to from Venice's unified <strong>Database</strong></li></ul> | 
| <strong>Pipeline</strong> | Source code adhering to our protocol for reading or writing data into or out of Venice, including the schema for how data is transformed to be compatible with external an <strong>Resource</strong> | <ul><li>Plaid <strong>Pipeline</strong> enables Venice to read data from Plaid's API</li><li>Quickbooks <strong>Pipeline</strong> enables Venice to read data from Quickbooks</li></ul> |
| <strong>Database</strong> | The primary data store connected to Venice where all of your unified data is stored. | A PostgreSQL <strong>Database</strong> configured to act as Venice's primary data store |
| <strong>API</strong> | A developer interface into Venice's unified <strong>Database</strong> that can be called to read or write data from or to Venice | <ul><li>A URL which can return data from Venice in JSON format</li><li>A URL which can return data from Venice in CSV format</li></ul> |
| <strong>Export</strong> | A manual mechanism through which data can be downloaded from Venice's unified <strong>Database</strong> (e.g. powered by our <strong>API</strong>) | You filter transactions data stored within Venice and download it to a local CSV file |
